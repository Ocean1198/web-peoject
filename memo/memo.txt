# 기본적인 개념 정리 용도.
# 이후에 삭제 예정


하나의 웹사이트를 위해 html, css, js 3개의 파일을 사용
각각의 역할은 다음과 같다.

html : 웹페이지의 기본 구조를 구성.
css  : 웹페이지의 모양이나 테마를 구성.
js   : 웹페이지의 동작이나 기능을 구성.


html의 구조는 다음과 같다.
기본 뼈대
    !DOCTYPE: 문서 유형 정의.
    <html>  : 문서 전체를 감싸는 최상위 태그
head
    title   : 제목 구성
    css와의 파일 연결
body
    <h1>    : 기본 텍스트
    <input> : 입력
    <button>: 버튼
    <script>: js의 실행

현재 js에서는 다음과 같은 코드줄이 있다.
const button = document.querySelector("button");
>> html 문서(document)에서 button 태그를 찾고, 이를 button에 저장.


html의 태그는 기본적으로 정의된 것을 사용, 기본 태그에 class나 id를 지정하는 형식을 쓴다.
기본 태그는 다음과 같다.

태그        의미
h1 ~ h6     제목
p           문단
button      버튼
input       입력
ul / li     목록

div / span  큰 의미 X(CSS,JS를 위한 사용)

canvas	    그림/게임
img	        이미지
video	    영상
audio	    소리


github <--> 내 컴퓨터(로컬) <--> VSC
가져오는 것은 pull, 내보내는 것은 push.
commit을 이용하게 되는데, commit은 일종의 기록과 같다.
- 의미 있는 변화가 있을 때
- 어떤 변화가 있는지를 설명.
작게, 자주 하는 것이 좋으며, 메시지로 변화를 기록한다.
저장의 개념보다는 기록의 개념으로, 이후에 메시지를 바탕으로 무엇을 진행했는지 알 수 있어야 한다.
* 메시지의 구성은 동사 + 명사 형식으로 하는 것이 기본적이고, 간단하며 명확하다.
